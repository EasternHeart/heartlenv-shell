Description

The panel library provides version information, which could be used by plugins to handle new API.

Details

libheartlenvpanel_major_version

extern const guint libheartlenvpanel_major_version;

The major version number of the libheartlenvpanel library (e.g. in version 4.8.0 this is 4).

This variable is in the library, so represents the libheartlenvpanel library you have linked against. Contrast with the LIBXFCE4PANEL_MAJOR_VERSION macro, which represents the major version of the libheartlenvpanel headers you have included.

libheartlenvpanel_check_version ()

const gchar *       libheartlenvpanel_check_version         (guint required_major,
                                                         guint required_minor,
                                                         guint required_micro);

Checks that the libheartlenvpanel library in use is compatible with the given version. Generally you would pass in the constants LIBXFCE4PANEL_MAJOR_VERSION, LIBXFCE4PANEL_MINOR_VERSION and LIBXFCE4PANEL_MICRO_VERSION as the three arguments to this function; that produces a check that the library in use is compatible with the version of libheartlenvpanel the extension was compiled against.

Example 1. Checking the runtime version of the Libheartlenvpanel library

const gchar *mismatch;
mismatch = libheartlenvpanel_check_version (LIBXFCE4PANEL_MAJOR_VERSION,
                                     LIBXFCE4PANEL_MINOR_VERSION,
                                     LIBXFCE4PANEL_MICRO_VERSION);
if (G_UNLIKELY (mismatch != NULL))
  g_error ("Version mismatch: %s", mismatch);


required_major :

 the required major version.

required_minor :

 the required minor version.

required_micro :

 the required micro version.

Returns :

 NULL if the library is compatible with the given version, or a string describing the version mismatch. The returned string is owned by the library and must not be freed or modified by the caller.
